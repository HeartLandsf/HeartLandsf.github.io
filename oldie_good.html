Purpose

In order to relieve development engineer from manually doing tedious and time consuming regression test before each release, basic test automation can help reducing a lot of manual labor. In addition to that, test automation will improve test coverage, ensure consistency and allow testing to happen more frequently.

This document is NOT to define test cases. Test cases are defined in the two documents “Windows_EUT_TestPlan.doc” and “88SE6440 –MV64XX EUT Test Plan and Report”. For each test case that can be automated, a corresponding Perl program will be written and a configuration file will be defined. The configuration file can be modified by user to fine tune the test case. Perl program will then base on the configuration file to run the test case and generate report.

More complete test automation such as installing OS, installing required tools, auto run a test script (when system is booted up) is also possible by using a server to control it as currently done by TAT.
